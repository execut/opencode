# Пессимистичная блокировка (pessimistic-locking)

Обязательные опции: нет

Предназначена для исключения ситуации конкурентной записи. Когда агрегат читается из БД, меняется в параллельном потоке и его изменения перезаписываются в текущем.
В таблицу необходимо добавить поле version. По-умолчанию оно 0.
При обновлении записей необходимо делать инкремент значения версии прямо в запросе: `UPDATE ... SET version = version + 1, остальные поля WHERE version=$version`.
Где $version - это версия, полученная при чтении агрегата.
В случае, если после обновления не поменялось ни одной записи, нужно пробрасывать ошибку, что запись уже поменялась другим запросом.

## В комбинации с другими опциями
### value-objects
Версия заворачивается в VO Version, создать который невозможно при отрицательной версии.

### ddd-light
У агрегата появляется новое поле с версией.
- При вызове AddAndApplyEvent в сущности, необходимо передавать следующую версию и при UPDATE делать минус 1 от неё. Например:
`UPDATE ... SET version = version + 1, остальные поля WHERE version=$version`, где $version - это версия из события минус 1.

# Состояния (states)

Обязательные опции: ddd-light

Все свойства сущностей переезжают в специальный DTO `<Entity>State`, где `<Entity>` - название сущности.
- Теперь, чтобы получить значение свойства сущности, её геттер обращается к State и пробрасывает его свойство.
- Ещё у сущности появляется метод State() <Entity>State. С его помощью репозиторий может доставать 

## business-events
Теперь в событиях есть свойство с интерфейсом State interface{}. Оно передаётся в конструктор NewEvent.

# Бизнес-события (business-events)
Обязательные опции: ddd-light

Бизнес-события (Domain Event) - это событие, описывающее факт в домене. С помощью событий агрегаты перемещают своё состояние в инфраструктурный слой.
На бизнес-события могут подписываться подписчики из разных слоёв через паттерн издатель-подписчик.

Во время действий над собой, агрегат вместо прямой модификации своих свойств, меняет их через события.
Заодно репозиторий может получить необходимые изменения через возникшие события, а не чтение свойств агрегата напрямую.

Появляются следующие файлы и папки:
- `<context>/` — папка ограниченного контекста или приложения, где <context> - название контекста или приложения
  - `domain/`
    - `event.go` - список возникающих бизнес-событий

- Для упрощения тест live-цикла не должен тестировать события и их методы. Тест должен проверять только изменения в состоянии сущности.
  - Каждый подтест цикла должен тестировать конкретное действие. Различные особые случаи с ошибками должны тестироваться вне live-цикла.
- В конструкторе сущности должен быть только идентификатор и внешние зависимости. Создание сущности должно быть только через её метод Create и порождение соответствующего события.
- Агрегаты и сущности теперь должны реализовывать интерфейс Entity

## Принципы
- Если бизнес-действие невалидно для текущего состояния агрегата, агрегат не должен порождать событие и должен вернуть ошибку, описывающую нарушенный бизнес-инвариант.

## В комбинации с другими опциями
### repository
- Репозиторий теперь имеет такой интерфейс:
```go
interface Repository {
    Load(ctx context.Context, entity *Entity) error // репозиторий добывает и применяет все события через AddAndApplyEvent и затем вызывает CleanEventList
    Save(ctx context.Context, entity *Entity) error // чистит события через CleanEventList и сохраняет текущее состояние сущности в БД. При необходимости пишет из них историю.
    // другие вспомогательные методы
} 
```
- Тип событий определяется через switch event.Payload().(type)
- События нельзя создавать через отдельные конструкторы. Нужно использовать только конструктор NewEvent

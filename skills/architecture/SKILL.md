# Архитектурные принципы
Этот скилл даёт описание архитектурных принципов и описывает опции архитектуры.
Вместе со скиллом загружай все инструкции из папки `./options`. Она находится в той же папке, где и текущий файл.

## Когда применять
Всегда, когда необходимо продумать реализацию кода

## Архитектурные опции
Архитектуру в зависимости от задач можно усложнять различными опциями. Такие опции перечислены в папке `options/`.
Перечень опций, которые нужно использовать в контексте, должны быть перечислены в файле `README.md`, в разделе "Используемые архитектурные опции".
Реализация считается корректной только если код явно отражает все архитектурные опции из `README.md`. Если задача кажется слишком простой для применения какой-либо опции, всё равно нужно применить её минимально корректным способом или остановиться и объяснить конфликт.
Если они не перечислены или такого раздела нет, предложи пользователю добавить и прекрати работу. 
Использовать опции, которые не перечислены в этом разделе ЗАПРЕЩЕНО!
Если возникла необходимость применять опцию не из этого раздела, необходимо сказать об этом пользователю и прекратить работу.
Пример структуры контекста:
```
- <context> - папка контекста с названием <context>.
- <context>/README.md - описание контекста и его опций. Идентифицируется заголовком "Контекст <context>"
```

### Структура опции
Каждый файл с опцией имеет следующую структуру:
- Название - название опции с алиасом опции
- Обязательные опции - перечень алиасов опций, которые должны включены в проекте. Без них нельзя использовать опцию. Если эти опции не были подключены в проекте, необходимо сообщить об этом пользователю и прекратить работу.
- Принципы - принципы, которых обязательно нужно придерживаться в проекте. При уходе от них, необходимо спросить разрешение у пользователя.


### Общий код опций
В папке `options/pkg/` есть общий код для опций. Необходимо её скопировать в папку `pkg` проекта.
В случае если в проекте уже есть папка pkg, необходимо её актуализировать и перезаписать одноимённые папки.
Внутри неё есть подпапки, совпадающие по названию с опциями архитектуры.
Там находится общий код для функционирования опций. Папки отражают вложенность опций: каждый уровень добавляет опцию. Например:
- `pkg`
  - `optimistic-locking` - опция оптимистичных блокировок
    - `busines-events` - опция оптимистичных блокировок в комбинации с бизнес-событиями
      - `value-objects` - опция оптимистичных блокировок в комбинации с бизнес-событиями и объектами-значениями

Комбинации приоритетней, чем основные варианты опций.

# Стиль кодирования

- **ВСЕГДА** Должны быть табы вместо пробелов
- Если объект реализует интерфейс, его необходимо проверять на соответствие этому интерфейсу через проверку по типу. Пример: `var _ contract.Application = (*App)(nil)`
- Каждый объект, его ошибки и конструкторы должны лежать в отдельном файле, названном по паттерну `<object_name>.go`, где `<object_name>` - название объекта в snake_case.
Семейство объектов нужно собирать в одном файле согласно названию основного родителя
- Всегда переменные с ошибками должны называться err, кроме случаев, когда нужно работать одновременно с двумя ошибками.
- Код нужно писать, подразумевая, что используется последняя версия go. Не нужно делать хаки с переменными. Например:
ТАК НЕ НУЖНО:
```
	for _, variant := range testApplicationVariants {
		variant := variant
```
ТАК НУЖНО:
```
	for _, variant := range testApplicationVariants {
		variant := variant
```
- В случае если в названии пакета несколько слов, их нужно разделять через _
- В случае если в названии пакета несколько слов, его в названии папки должны быть дефисы "-" вместо пробелов
- Конструктор должен располагаться сразу после объявления структуры, которую он создаёт. Например:
```
type Service struct {
    conn *pgx.Conn
}

func NewService(ctx context.Context) (*Service, error) {
    ...
}
```
- Не нужно объявлять переменные тогда, когда они присваиваются в следующей строчке. Например:
НЕЛЬЗЯ:
```go
var err error
product, err = domain.NewProduct(id, testhelper.TestProductName)
```
МОЖНО:
```go
product, err := domain.NewProduct(id, testhelper.TestProductName)
```
